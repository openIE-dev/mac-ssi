The flagship

Run a model no single Mac can hold.

One command shards a frontier model across every Mac's GPU memory and serves it as a single OpenAI endpoint. Point any client at it — it just sees one big model.

# serve a frontier model across the whole cluster
$ ssi serve nemotron-ultra-550b --port 8080

# it's just the OpenAI API — point any tool at it
$ curl http://localhost:8080/v1/chat/completions \
    -d '{"messages":[{"role":"user","content":"Why is the sky blue?"}]}'
from openai import OpenAI
client = OpenAI(base_url="http://localhost:8080/v1", api_key="local")
print(client.chat.completions.create(
    model="ultra-550b",
    messages=[{"role":"user","content":"What can a cluster of Macs do that one can't?"}],
).choices[0].message.content)

Proven on hardware: a 120B model distributed across 3 Macs at ~22 tok/s through one endpoint. Built for 400–600B-class models on pooled cluster RAM.

How it works

Closing the 250× bandwidth gap.

Apple's on-die UltraFusion runs at 2.5 TB/s; the external Thunderbolt link at 10 GB/s. mac-ssi bridges that gap in software — and pipeline-shards each model so a slice of the layers lives on each node, with only ~tens of KB of activations crossing the fabric per token.

API & CLI

Everything is one ssi command.

CommandWhat it does
ssi serve <model>Distributed inference of a model bigger than one node → one OpenAI endpoint
ssi run ./workloadRun any binary on the best node, scheduled by GPU / ANE / RAM / energy
ssi status · nodes · resourcesSee the cluster as one machine
ssi topologyFabric topology — the Thunderbolt links between nodes
ssi gpu · anePool and dispatch to GPU / Neural Engine across nodes
ssi memoryDistributed shared memory across cluster RAM
ssi fsOne virtual filesystem spanning every node's storage
ssi ps · killManage processes cluster-wide
